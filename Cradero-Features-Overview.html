What is Cradero?

Cradero is a web-based maintenance management system (CMMS) designed for manufacturing and production environments. It centralises work order management, equipment tracking, preventive maintenance scheduling, parts inventory, reactive breakdown logging, and reporting into a single application.

The software is available in two tiers: ESSENTIAL and PRO.

Essential Tier Features

1. Work Order Management

  • Create, edit, and delete work orders with full detail capture
  • Priority levels — High, Medium, Low — with visual colour coding
  • Status tracking — Open, In Progress, On Hold, Completed
  • Engineer assignment — assign or reassign to any team member
  • Photo attachments — upload photos on creation and add more photos from the detail view at any time
  • Parts usage tracking — book out parts to work orders with automatic stock deduction and low-stock warnings
  • Return parts to stores — remove booked parts if unused, automatically restoring inventory stock levels
  • Comments thread — timestamped comments from any team member
  • Time tracking — start time, finish time, auto-calculated total job time
  • Machine downtime logging — record downtime per work order
  • Search & filter — full-text search, filter by status, priority, and sort by date or priority

2. Equipment Asset Management

  • Full equipment register — name, ID, manufacturer, model, location, install date
  • Criticality ratings — A (Critical), B (Important), C (Low)
  • Status tracking — Operational, Maintenance, Down
  • Warranty management — expiry date, provider, notes, with automatic status badges (Active / Expiring / Expired)
  • Equipment history — maintenance statistics, compatible parts, and full info panel
  • Compatible parts linking — see which inventory parts fit each machine
  • Add, edit, delete any equipment record

3. Parts Inventory

  • Full parts register — name, part number, quantity, minimum stock level, shelf/bin location
  • Stock status indicators — In Stock, Low Stock, Out of Stock (auto-calculated)
  • Monthly usage tracking — records how many times each part has been used
  • Book out to jobs — parts can be booked out to both work orders and PM tasks, with automatic stock deduction
  • Return to stores — unused parts can be returned, automatically restoring stock levels
  • Compatible equipment linking — associate parts with specific machines
  • Edit and delete parts
  • Reorder functionality — order more stock directly from the parts screen

4. Preventive Maintenance (PM)

  • Schedule PM tasks with equipment, frequency (Daily through Annual), duration, and assigned engineer
  • Checklists — step-by-step checks that must all be ticked before completion
  • Per-item notes — engineers add notes to individual checklist items during completion
  • Start/finish time capture on completion
  • Parts usage on completion — book out parts during PM completion with automatic stock deduction and low-stock alerts
  • Raise work order from PM — if issues are found that cannot be fixed at the time, raise a work order directly from the PM completion screen with issue title, description, and priority
  • Manager/Admin editing — managers and admins can edit PM tasks including title, description, frequency, duration, assigned engineer, equipment, and add or remove checklist items
  • Manager/Admin photo upload — managers and admins can attach photos during PM completion for documentation
  • Due status tracking — Overdue, Due Soon, Upcoming, Completed
  • Reassignment — reassign PM tasks to different engineers from the list
  • Filters — by status (Complete / Scheduled / Unscheduled), equipment, and date range
  • Audit trail — view completed checklist results, notes, parts used, and photos from previous completions

5. Daily Activity Log

  • Timeline view of all maintenance activity for a selected date
  • Activity types — work orders created, started, completed; PM tasks completed
  • Metadata — engineer name, duration, parts used
  • Daily summary KPIs — work orders created, completed, PM tasks completed, total downtime

6. Reactive Maintenance / Breakdown Logging

  • Log breakdowns with machine, date/time, fault description, root cause, and resolution
  • Status tracking — Resolved, In Progress, Pending
  • Auto-raise work orders — optionally create a linked work order from a breakdown
  • Resolution recording — document the fix applied and time to resolve
  • Production loss tracking — record hours of production lost per breakdown
  • Filters — by machine, status, and date range
  • Edit and delete breakdown records

7. My Tasks / Engineer Dashboard

  • Per-engineer view — select an engineer to see their assigned work
  • Work orders section — all open/in-progress work orders assigned to that engineer
  • PM tasks section — assigned PMs with due status and quick-complete button
  • Auto-selects current user on login

8. Work Requests

  • Submit maintenance requests from the shop floor — select equipment, describe the issue, set urgency
  • Urgency levels — Low, Medium, High
  • Approval workflow — managers approve (auto-creates work order) or decline
  • Status tracking — Pending, Approved, Declined
  • Filter by status and urgency

9. Notifications

  • Notification bell with unread count badge
  • Alert categories — overdue PMs, unassigned work orders, expiring permits, completed WOs, warranty expirations
  • Click-to-navigate — clicking a notification takes you to the relevant item

10. Dark Mode

  • Toggle dark mode from the header with one click
  • Full theme support — all components, modals, charts, and cards adapt to dark theme
  • Preference saved per user session

11. Role-Based Access Control

  • Admin — full access to all features, approve new users, manage all data
  • Manager — approve work requests, approve purchase orders, edit PM tasks (add/remove checklist items, change frequency, reassign), attach photos to PM completions, delete records
  • Engineer — create/edit work orders, complete PM tasks, log breakdowns, book out parts, submit work requests
  • Pending — newly registered users are blocked until admin approval

12. Mobile Responsive Design

  • Fully responsive layout — works seamlessly on phones, tablets, and desktops
  • Adaptive navigation — sidebar collapses to a mobile-friendly menu on smaller screens
  • Touch-optimised controls — buttons, modals, and forms are sized for touch interaction
  • On-the-go access — engineers can create work orders, complete PMs, log breakdowns, and upload photos directly from the shop floor
  • No app download required — runs in any mobile browser with no installation needed

Pro Tier Features

Everything in Essential, plus the following:

13. Purchase Orders

  • Create purchase orders with vendor, line items, quantities, unit prices, auto-calculated totals
  • Approval workflow — pending POs require manager approval before being sent
  • Status tracking — Pending, Approved, Ordered, In Transit, Received, Cancelled
  • Quick filters — filter POs by status with one click
  • Tracking numbers — record and display shipping tracking info
  • Budget codes — assign POs to budget categories (Maintenance, Emergency, Capital, Line Operating)
  • Spending analysis — breakdowns by vendor, category, and budget code
  • Summary KPIs — open POs, total value, monthly spending, YTD budget usage
  • Auto-PO creation — ordering parts from inventory automatically creates a purchase order

14. Permits Management

Hot Work Permits

  • Issue permits for welding, cutting, grinding, or any spark-producing work
  • Safety checklist — fire extinguisher, area cleared, fire watch assigned, equipment inspected, emergency procedures
  • Time-limited — start/end times with expiry countdown
  • Close permits when work is complete

Contractor Work Permits

  • Issue permits for external contractors working on-site
  • Capture company name, supervisor, phone, worker count, insurance certificate
  • Compliance checklist — safety orientation, insurance, emergency procedures, PPE, tools inspected
  • Scope of work documentation
  • View details and revoke permits

15. Analytics & Reports

Interactive Dashboards

  • Equipment Downtime by Line — bar chart showing hours per production line
  • Work Order Status Distribution — doughnut chart (Open / In Progress / Completed)
  • PM Compliance Trend — line chart showing compliance % over 6 months

Advanced Metrics

  • Top 5 Problem Equipment — ranked by work order count (30 days)
  • Maintenance Cost Breakdown — labour, parts, contractor costs
  • PM Completion Rate — on-time, late, overdue, with compliance %
  • Parts Stock Forecast — parts at risk of depletion with months remaining
  • MTTR & MTBF — Mean Time to Repair / Mean Time Between Failures per machine
  • Technician Workload — active WOs and PMs per engineer
  • Equipment Health Scores — composite 0–100 score per machine
  • Labour Costs — calculated from tracked time × hourly rates

CSV Exports

  • Work Orders, Equipment, Parts Inventory, Breakdowns, PM Schedule, Purchase Orders
  • Full Maintenance Report — comprehensive export of all KPIs and data

16. Maintenance Calendar

  • Month view — full calendar grid with colour-coded event pills
  • Week view — 7-day column layout with detailed event cards
  • Event types — Weekly PM (blue), Monthly PM (purple), Major Service (red), Inspection (yellow), Shutdown (dark red)
  • Add planned jobs — title, equipment, type, date, duration, assigned engineer
  • Navigate between months/weeks with arrow buttons

17. Supplier Management

  • Multiple suppliers per part — supplier name, part number, unit price, lead time
  • Supplier comparison panel — side-by-side pricing and lead times when ordering
  • Smart vendor dropdown — pre-populated from part supplier data

18. Equipment QR Codes

  • Generate QR codes for any equipment asset directly from the equipment card
  • Printable — one-click print to label printers or standard printers
  • Encoded data — contains equipment ID for quick scanning and lookup

19. Audit Trail

  • Full audit log of all create, update, delete, and complete actions
  • Filterable by action type, entity type, and date range
  • Tracks who did what, when, and to which record
  • Parts returns logged when parts are returned to stores

20. Offline Mode

  • Service Worker for offline capability
  • Cached authentication — app works when internet is unavailable
  • Offline indicator — "OFFLINE" badge displayed in header
  • Full local data persistence — all functionality continues offline

21. Equipment SOP Documents

  • Attach SOPs — upload PDF, Word, or image files directly to any equipment asset
  • View & download — SOPs accessible from the equipment detail view with one-click open
  • Manage documents — add or remove SOPs at any time from the equipment edit screen
  • Cloud storage — all documents stored securely in Supabase Storage
  • On-the-floor access — engineers can reference SOPs from any device while working on equipment

22. Manufacturer Manuals

  • Attach manuals — upload manufacturer manuals (PDF, Word, images) to any equipment asset
  • Instant access — engineers can view and download manuals from the equipment detail screen on any device
  • Multiple documents — attach as many manuals as needed per machine (installation guides, service manuals, parts diagrams)
  • Manage documents — add or remove manuals at any time from the equipment edit screen
  • No more searching — all manufacturer documentation lives alongside the equipment record, accessible in seconds

23. Production / QA User Role

  • Dedicated dashboard — production and QA staff see only the Work Requests tab
  • Submit requests — report equipment issues with equipment selection, description, and urgency level
  • Track status — view real-time status of all submitted requests (Pending, Approved, Declined)
  • No maintenance access — work orders, parts, PMs, and other maintenance modules are hidden
  • Clean separation — production staff report issues, maintenance engineers resolve them

Key Workflows

Work Order Lifecycle

  1. Work order created (manually, from work request approval, from breakdown, or raised during PM completion)
  2. Assigned to engineer
  3. Engineer starts work — status moves to In Progress, start time recorded
  4. Parts booked out from inventory as they are used (with automatic stock deduction)
  5. Additional photos attached from the detail view as work progresses
  6. Engineer adds comments and tracks time
  7. Unused parts returned to stores (stock automatically restored)
  8. Work completed — finish time recorded, total job time auto-calculated
  9. Machine downtime logged

PM Completion Workflow

  1. PM task becomes due (overdue / due-soon badge appears)
  2. Engineer clicks “Complete Now”
  3. Start time pre-filled, each checklist item must be ticked off
  4. Optional notes added per checklist item
  5. Parts used during the PM can be booked out from inventory
  6. If issues are found, “Raise Work Order” option creates a linked WO with issue details
  7. Managers/admins can attach photos for documentation
  8. Finish time entered
  9. All items verified — task marked complete, parts deducted from stock
  10. Next due date auto-calculated based on frequency

PM Completion → Work Order Flow

  1. During PM completion, engineer discovers an issue that cannot be fixed immediately
  2. Engineer ticks “Raise Work Order for issues found?”
  3. Enters issue title, description, and priority
  4. On PM submission, work order is auto-created with the PM’s equipment pre-filled
  5. Work order appears in the main work orders list for assignment and tracking

Breakdown → Work Order Flow

  1. Breakdown logged in Reactive Maintenance
  2. “Also raise a Work Order” checkbox auto-checked for unresolved issues
  3. Work order auto-created with high priority, linked to breakdown
  4. When breakdown is resolved, linked work order auto-completed

Work Request → Work Order Flow

  1. Floor worker submits maintenance request with equipment, description, urgency
  2. Manager reviews in Work Requests tab
  3. Manager selects engineer and clicks “Approve & Create WO”
  4. Work order auto-created with request details, assigned to selected engineer

Security

Authentication & Session Management

  • Supabase Auth — industry-standard email/password authentication with secure token handling
  • Session timeout — automatic logout after 30 minutes of inactivity (click, keydown, or touch — not passive mouse movement)
  • Throttled session reset — timer only resets once per 60 seconds to prevent phantom sensor activity keeping sessions alive
  • Pending user approval — new sign-ups are blocked from accessing the application until an admin approves their account
  • Password requirements — minimum 6 characters with confirmation field to prevent typos

Data Protection

  • XSS prevention — all user-supplied data is escaped via escHtml() before rendering to the page, preventing cross-site scripting attacks
  • Input sanitisation — dedicated sanitizeInput() function trims and truncates all user input to enforced character limits
  • Input length limits — field-specific maximums enforced (titles: 200 chars, descriptions: 2,000 chars, comments: 1,000 chars, names: 150 chars)
  • Auto-applied maxlength — all text inputs and textareas automatically have maxlength attributes set on page load
  • Safe file naming — uploaded photo filenames are sanitised to remove special characters before storage

Access Control

  • Role-based permissions — Admin, Manager, Engineer, and Pending roles control what each user can see and do
  • Delete protection — only managers and admins can delete records (work orders, equipment, parts, PMs, breakdowns)
  • PM edit protection — only managers and admins can edit PM task definitions (checklists, frequency, assignment)
  • Tier-based feature gating — Pro-only features are hidden and inaccessible on Essential tier accounts
  • IIFE encapsulation — all application logic runs inside an Immediately Invoked Function Expression; internal data variables (user role, tier, work order data, etc.) cannot be accessed or tampered with via browser DevTools
  • Controlled function exposure — only functions needed by HTML event handlers are explicitly exposed to the global scope via the _expose array

Audit & Accountability

  • Full audit trail — every create, update, delete, and complete action is logged with timestamp, user, entity type, and entity ID
  • Parts tracking — all parts booked out and returned to stores are logged with quantities and associated job references
  • Created-by tracking — work orders, breakdowns, and other records store who created them
  • PM completion history — checklist results, notes, parts used, and photos from every PM completion are permanently recorded
